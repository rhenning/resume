To create a GitHub Actions workflow that converts Markdown files to PDF, RTF, and DOC files, and then generates a GitHub release artifact, we can break the task into a few steps:

  1. Set up the Workflow: Trigger the workflow on push or pull request events for a Markdown file.
  2. Convert Markdown to PDF, RTF, and DOC: Use tools like Pandoc (a powerful document converter) to convert the Markdown file into these formats.
  3. Create Release Artifacts: After generating the files, upload them as release assets using GitHub Actions' actions/upload-artifact and gh CLI for creating the release.

Example Workflow

Here’s an example of a GitHub Actions workflow that will run on the main branch, convert a Markdown file into PDF, RTF, and DOCX formats, and then upload those files as release assets.

yaml
name: Convert Markdown to PDF, RTF, and DOCX

on:
  push:
    branches:
      - main
    paths:
      - '**.md'  # Trigger the workflow when a markdown file is pushed

jobs:
  convert_markdown:
    runs-on: ubuntu-latest

    steps:
    - name: Checkout code
      uses: actions/checkout@v3

    - name: Set up Pandoc
      run: sudo apt-get install -y pandoc texlive-xetex

    - name: Install LibreOffice (for DOCX and RTF support)
      run: sudo apt-get install -y libreoffice

    - name: Convert Markdown to PDF
      run: pandoc input.md -o output.pdf

    - name: Convert Markdown to RTF
      run: pandoc input.md -o output.rtf

    - name: Convert Markdown to DOCX
      run: libreoffice --headless --convert-to docx input.md

    - name: Create a GitHub release
      run: |
        VERSION_TAG="v$(date +'%Y%m%d%H%M%S')"  # Use a timestamp as version tag
        gh release create "$VERSION_TAG" \
          output.pdf output.rtf output.docx \
          --title "Markdown Conversion Release" \
          --notes "Converted markdown file to PDF, RTF, and DOCX formats."

    - name: Upload generated files as artifacts (optional)
      uses: actions/upload-artifact@v3
      with:
        name: markdown-conversion-files
        path: |
          output.pdf
          output.rtf
          output.docx

Explanation

  1. Triggering the Workflow:

    • The workflow is triggered by any push event to the main branch that affects Markdown files (**.md).

  2. Setup Steps:

    • Checkout the Code: The actions/checkout@v3 action checks out your repository so that the workflow can access the Markdown file.
    • Install Pandoc: We install Pandoc and TeX Live (needed for PDF conversion) to convert the Markdown file to different formats.
    • Install LibreOffice: LibreOffice is installed to handle converting the Markdown file to DOCX and RTF formats.

  3. Conversion Steps:

    • We use pandoc to convert the Markdown (input.md) to PDF and RTF.
    • We use LibreOffice (in headless mode) to convert the Markdown file to DOCX.

  4. Release Creation:

    • The gh release create command creates a new GitHub release with the converted files as assets. The release is tagged with the current timestamp (as a simple version tag).

  5. Optional Artifact Upload:

    • You can also choose to upload the converted files as artifacts for other jobs or further use.

Requirements

  1. Markdown File: The workflow expects a Markdown file (input.md) to be in the repository’s root directory. You can modify the file path in the pandoc and LibreOffice commands if your Markdown file is located elsewhere.

  2. GitHub CLI: The gh CLI must be set up with the necessary permissions for creating releases. The action will require permissions to create releases, so make sure the GitHub Actions runner has access to the appropriate repository scopes.

  3. Pandoc and LibreOffice: Both tools are installed in the Ubuntu runner to handle the conversion. Pandoc works for most formats, but LibreOffice is required for DOCX and RTF conversions due to the complexity of these formats.

Notes:

  • If you want to trigger the workflow for a specific file (like README.md), you can adjust the paths field accordingly in the on section.
  • The gh CLI command assumes the repository is configured for use with GitHub Actions. Ensure that the GitHub CLI is authenticated to create a release. You can use a GITHUB_TOKEN secret if necessary.

This workflow will automatically generate a release with the converted documents every time you push a new Markdown file to the main branch.
